# Social & Community Features

Bingeki transforms solo tracking into a social experience. This document outlines the community-driven features and their underlying data models.

## Global Activity Feed

The feed is a real-time stream of what's happening in the community.

### 1. Activity Types
*   **Manga/Anime Interaction**: "Sasuke marked *Hunter x Hunter* as Completed."
*   **XP Millestones**: "Naruto reached Level 10!"
*   **Social Interactions**: "Luffy is now friends with Zoro."
*   **System Events**: New news articles or site updates.

### 2. Implementation
Activities are generated by **Firestore Triggers** (`onLibraryUpdate`). This ensures the feed is always accurate and cannot be manually manipulated by clients.
*   **Storage**: `/activities/{activityId}`
*   **Permission**: Users can read the feed, but only the Admin SDK can write to it.

## Friendship System

Bingeki uses a "Mutual Acceptance" model for friendships.

### 1. Request Lifecycle
1.  **Pending**: User A sends a request to User B. A "Friend Request" document is created in `/users/{B}/requests`.
2.  **Accepted**: User B accepts. A Cloud Function removes the request and adds User A to `/users/{B}/friends` and User B to `/users/{A}/friends`.
3.  **Real-Time Sync**: Profile snapshots (name, avatar, level) are stored directly in the friends list to avoid expensive cross-document joins during list rendering.

## Leaderboards & Rankings

Compete with friends or the entire world.

### 1. The Podium
The top 3 global hunters are displayed prominently with a "Gold/Silver/Bronze" visual theme (see `src/components/social/Podium.tsx`).

### 2. Global Ranking List
A paginated list of all users sorted by `totalXp`.
*   **Performance**: Rankings are indexed in Firestore for sub-second retrieval.
*   **Friend-Only Filtering**: Users can toggle the leaderboard to see only their friends' standings.

---

## Profile Sharing

Every hunter has a unique public profile URL:  
`bingeki.app/profile/{username}`

### 1. Dynamic OG Images
When sharing a profile link on Discord or Twitter, a custom "Hunter License" card is generated on-the-fly via Cloud Functions.
*   Shows Username, Level, and Rank.
*   Displays the user's "Signature" anime/manga.

### 2. Privacy Controls
Users can choose to make their library:
*   **Public**: Anyone can see it.
*   **Friends Only**: Only accepted friends can view.
*   **Private**: Visible only to the owner.
